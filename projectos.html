The problem

Large language models are excellent conversational collaborators, but conversation is not a reliable execution substrate. A model can understand a request, discuss the correct protocol, and still fail to perform the required reads, writes, verification, or handoff. Context disappears across threads. Identity drifts. Completion claims outpace durable evidence.

The shape

ProjectOS models the environment as a control plane containing Nodes, lifecycle commands, authority, workspaces, relays, durable state, execution artifacts, and evidence. Nodes are workers. Repositories are durable memory. IPC carries inter-Node messages. POSL expresses human-readable intent that can be compiled into bounded execution packets.

Conversation remains the interface. Deterministic artifacts become the execution contract.

Implemented architecture

  • Canonical Node identity and alias resolution.
  • Generic startup and shutdown procedures with Node-specific overlays.
  • Repository-backed state, handoffs, relays, ACKs, and lifecycle receipts.
  • Authority and operational-lethality boundaries.
  • POSL-HL parsing, AST creation, lowering, execution-packet generation, and packet-integrity verification.
  • Stage-0 authoritative runtime path with mutation blocked before execution when integrity fails.
  • Local and hosted control-plane service slices with adversarial validation.
  • Workspace binding that separates who a Node is from the repository where it currently works.

Why it matters

Most agent frameworks concentrate on how models call tools. ProjectOS concentrates on how an organization can know which Node acted, under what authority, against which durable state, with what evidence, and whether the work can be resumed safely by another session.

Current frontier

The current compiler surface intentionally remains narrow while the platform hardens. Near-term work includes profile-generic authoritative startup, workspace activation, object-aware evidence references, and eventually deterministic R4U and shutdown execution.

Architectural boundary: ProjectOS coordinates work; it does not absorb the production runtimes or domain models of the systems its Nodes build.
